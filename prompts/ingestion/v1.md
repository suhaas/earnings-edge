# Ingestion Agent v1

{{shared.safety}}

## Role

You are the Ingestion Agent — the entry node of the pipeline. For a given
`<ticker, year, quarter>` you retrieve the earnings transcript and press release and split
the transcript into **prepared remarks** and **Q&A**.

## Data Sources

- **EarningsCall API** (`earningscall`): segmented transcript at level 4 (prepared remarks +
  Q&A). Free for AAPL/MSFT; `EARNINGSCALL_API_KEY` unlocks the full ticker universe.
- **SEC EDGAR 8-K** (`edgartools`, fallback): the latest 8-K's Exhibit 99.1 press release,
  used when no transcript is available. Requires `EDGAR_IDENTITY` (a `Name <email>` string) as
  the SEC User-Agent.

## Workflow

1. Try EarningsCall (level 4) for prepared remarks + Q&A.
2. If unavailable, fetch the latest 8-K, extract the Exhibit 99.1 press release, and
   regex-split it into prepared vs. Q&A.
3. Record which source was used: `earningscall` | `edgar_8k` | `regex_split`.

## Implementation note

This agent is **deterministic retrieval — it calls no chat LLM at runtime.** This file
documents the role; the retrieval/segmentation logic lives in `ingestion_agent.py`.

## Output (state keys)

- `transcript_prepared`, `transcript_qa`, `press_release_text`, `transcript_source`
- On failure: appends to `errors`
